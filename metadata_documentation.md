## 🏷️ الحقول التعريفية الموحدة (Schema Metadata)
تحتوي كافة ملفات الـ JSON في المجلد على كائن أبوي (Root Object) يحتوي على الحقول التعريفية الموحدة التالية لإدارة التوافق:
* `schemaVersion` (Integer): إصدار مخطط البيانات (النسخة الحالية: `1`).
* `lastUpdated` (String - ISO 8601): تاريخ ووقت آخر تحديث للملف (مثل: `2026-07-19T01:57:00Z`).
* `generatedBy` (String): معرّف النظام المنشئ للبيانات (مثل: `Alpha Manager Build System`).

*ملاحظة:* للملفات التي تحوي مصفوفات (Lists)، يتم تضمين المصفوفة داخل كائن مغلف بحقل باسم `"data"` تحت الجذر، وذلك لضمان وجود الحقول التعريفية في جذر الملف بشكل موحد.

## 1. `update.json`
* **وظيفة الملف:** فحص وتطبيق التحديثات التلقائية بالبرنامج وتوفير بيانات صفحة التحميل الرئيسية للموقع.
* **الجهة المستهلكة:** كلا الطرفين (البرنامج والموقع الرسمي).

| الحقل | الوصف | نوع البيانات | جهة الاستخدام | قواعد التحقق |
| :--- | :--- | :--- | :--- | :--- |
| `version` | رقم الإصدار الجديد المتاح | String | كلاهما | يجب أن يتوافق مع نمط SemVer (مثل: `2.0.2`) |
| `updateType` | نوع التحديث ومدى أهميته | Enum | البرنامج | يقبل القيم: `optional`, `recommended`, `mandatory`, `security` |
| `mandatory` | تحديد هل الترقية إجبارية فوراً | Boolean | البرنامج | قيمة منطقية `true` أو `false` |
| `minimumSupportedVersion` | أدنى إصدار يُسمح بتشغيله | String | البرنامج | يجب أن يتوافق مع SemVer (مثل `2.0.2`) |
| `downloadUrl` | رابط التحميل المباشر لملف التثبيت | String (URL) | كلاهما | يجب أن يبدأ بـ `https://` ويكون رابطاً مباشراً لملف EXE |
| `sha256` | البصمة الرياضية لملف التثبيت | String | كلاهما | كود تشفير SHA256 بطول 64 حرفاً |
| `fileSize` | حجم الملف الفعلي بالبايت | Integer (Long) | كلاهما | رقم صحيح أكبر من الصفر |
| `releaseDate` | تاريخ إطلاق الإصدار | String (Date) | كلاهما | بالصيغة القياسية `YYYY-MM-DD` |
| `releaseNotes` | قائمة نصية بسيطة للميزات | Array of Strings | البرنامج | مصفوفة نصوص قصيرة غير فارغة |
| `message` | رسالة إدارية قصيرة تظهر للمستخدم | String | البرنامج | نص لا يتجاوز 250 حرفاً |
| `enabled` | حالة التحديث (هل هو نشط حالياً؟) | Boolean | البرنامج | قيمة منطقية `true` أو `false` |
| `extendedInfo` | كائن يحتوي على تفاصيل موسعة للموقع | Object | الموقع | كائن يحتوي على الحقول التفصيلية للموقع |
| `structuredReleaseNotes` | قائمة ملاحظات إصدار مصنفة وهيكلية | Object | الموقع | كائن يحتوي على قوائم مصنفة للإضافات والإصلاحات وغيرها |

## 2. `versions.json`
* **وظيفة الملف:** الاحتفاظ بسجل وتاريخ الإصدارات لعرض صفحة "أرشيف التنزيل والإصدارات السابقة".
* **الجهة المستهلكة:** الموقع الرسمي.
* **البنية الأساسية:** كائن يحتوي على الحقول التعريفية وحقل `"data"` الذي يحوي مصفوفة من الإصدارات السابقة.

### حقول عناصر مصفوفة `data`:
* `version` (String): رقم الإصدار المطروح (SemVer قياسي ومميز لا يتكرر).
* `buildNumber` (Integer): رقم البناء المقابل (رقم تصاعدي).
* `releaseDate` (String - Date): تاريخ طرح الإصدار بالصيغة القياسية `YYYY-MM-DD`.
* `downloadUrl` (String - URL): رابط مباشر للملف على السيرفر أو GitHub.
* `fileSize` (Integer): حجم الملف بالبايت.
* `sha256` (String): كود تشفير SHA256 بطول 64 حرفاً.
* `supportStatus` (Enum): يقبل `active` (مستقر)، `deprecated` (قديم)، `eol` (منتهي الدعم).
* `category` (String): يقبل `Major`, `Minor`, `Hotfix`.
* `releaseNotesSummary` (String): ملخص نصي قصير للتحديث.
* `releaseNotes` (Object): ملاحظات الإصدار المصنفة (تحوي مصفوفات `additions`, `improvements`, `fixes`, `security`).

## 3. `screenshots.json`
* **وظيفة الملف:** تغذية معرض لقطات الشاشة والصور التوضيحية لواجهات البرنامج بشكل تفاعلي وديناميكي.
* **الجهة المستهلكة:** الموقع الرسمي.
* **البنية الأساسية:** كائن يحتوي على الحقول التعريفية وحقل `"data"` الذي يحوي مصفوفة اللقطات.

### حقول عناصر مصفوفة `data`:
* `title` (String): عنوان اللقطة باللغة العربية (مثل: "لوحة التحكم الرئيسية").
* `description` (String): شرح مبسط لوظيفة الشاشة والميزات الظاهرة بها.
* `imageUrl` (String - URL): رابط آمن `https://` للصورة (JPG/PNG/WebP).
* `category` (String): تصنيف الصورة (مثل `General`, `Voucher Printing`, `Hotspot Management`).
* `displayOrder` (Integer): رقم تصاعدي للترتيب.
* `isFeatured` (Boolean): هل الصورة مميزة وتعرض في الصفحة الرئيسية؟

## 4. `features.json`
* **وظيفة الملف:** توليد محتوى صفحة "ميزات المنظومة" والبطاقات التسويقية والتقنية تلقائياً.
* **الجهة المستهلكة:** الموقع الرسمي.
* **البنية الأساسية:** كائن يحتوي على الحقول التعريفية وحقل `"data"` الذي يحوي مصفوفة الميزات.

### حقول عناصر مصفوفة `data`:
* `title` (String): عنوان الميزة الرئيسي.
* `description` (String): شرح تفصيلي لطبيعة وكيفية عمل الميزة وفائدتها.
* `category` (String): تصنيف الميزات (مثل `Security`, `Licensing`, `Hotspot Management`, `Voucher Printing`).
* `iconName` (String): اسم أيقونة مكتبة الأيقونات المستخدمة بالموقع (مثل `key-outline`).
* `isKeyFeature` (Boolean): هل الميزة رئيسية وتعرض بالصفحة الرئيسية للموقع؟
* `displayOrder` (Integer): ترتيب ظهور الميزة في الصفحة.

## 5. `requirements.json`
* **وظيفة الملف:** توفير معلومات تفصيلية ودقيقة لمتطلبات وبيئات التشغيل المطلوبة لتفادي الأخطاء.
* **الجهة المستهلكة:** الموقع الرسمي.

| الحقل | الوصف | نوع البيانات | قواعد التحقق |
| :--- | :--- | :--- | :--- |
| `supportedWindowsVersions` | قائمة إصدارات نظام ويندوز المتوافقة | Array of Strings | مصفوفة إصدارات نظام ويندوز المدعومة |
| `architecture` | المعماريات المتوافقة | String | نص توضيحي للمعماريات المتوافقة |
| `minimum` | الحد الأدنى من متطلبات التشغيل الأساسية | Object | كائن يحتوي على حقول المواصفات الدنيا (`cpu`, `ram`, `diskSpace`, `dotnetVersion`) |
| `recommended` | المواصفات والمكونات الموصى بها لأفضل أداء | Object | كائن يحتوي على حقول المواصفات الموصى بها (`cpu`, `ram`, `diskSpace`, `dotnetVersion`) |
| `additionalSoftware` | البرمجيات المساعدة المطلوب تثبيتها | Array of Strings | مصفوفة بالبرمجيات الخارجية اللازمة |

## 6. `faq.json`
* **وظيفة الملف:** إدارة وعرض الأسئلة الأكثر شيوعاً وإجاباتها الفنية لدعم العملاء ديناميكياً.
* **الجهة المستهلكة:** الموقع الرسمي.
* **البنية الأساسية:** كائن يحتوي على الحقول التعريفية وحقل `"data"` الذي يحوي مصفوفة الأسئلة.

### حقول عناصر مصفوفة `data`:
* `question` (String): نص السؤال (ينتهي بـ `؟`).
* `answer` (String): الإجابة التفصيلية الفنية على السؤال.
* `category` (String): تصنيف السؤال (مثل `Licensing`, `Compatibility`, `Features`, `General`).
* `displayOrder` (Integer): ترتيب ظهور السؤال.

## 7. `settings.json`
* **وظيفة الملف:** توفير ثوابت وإعدادات الموقع العام والتوجيه والترجمة لمنع التكرار والكتابة اليدوية.
* **الجهة المستهلكة:** الموقع الرسمي.

| الحقل | الوصف | نوع البيانات | قواعد التحقق |
| :--- | :--- | :--- | :--- |
| `appName` | اسم التطبيق والبرنامج الرسمي | String | نص قصير |
| `website` | رابط الموقع الإلكتروني الرسمي | String (URL) | يبدأ بـ `https://` |
| `supportEmail` | بريد الدعم الفني والمراسلة | String (Email) | صيغة بريد إلكتروني صالحة |
| `githubRepository` | رابط مستودع الأكواد الرئيسي | String (URL) | يبدأ بـ `https://` |
| `githubReleases` | رابط الإصدارات المطروحة على GitHub | String (URL) | يبدأ بـ `https://` |
| `privacyUrl` | رابط مسار سياسة الخصوصية | String | مسار نسبي أو مطلق (مثل `/privacy`) |
| `termsUrl` | رابط مسار الشروط والأحكام | String | مسار نسبي أو مطلق (مثل `/terms`) |
| `copyright` | نص حقوق الملكية الفكرية المعروض في التذييل | String | يحتوي على رمز الحقوق والترخيص |
| `defaultLanguage` | لغة الموقع الافتراضية للزوار | String | رمز اللغة المعياري (مثل `ar` أو `en`) |
| `theme` | المظهر الافتراضي للموقع | String | يقبل: `dark`, `light` |

## 8. `social.json`
* **وظيفة الملف:** تجميع كافة قنوات التواصل والدعم الفني وتغذية تذييل الموقع وأزرار الدعم.
* **الجهة المستهلكة:** الموقع الرسمي وأي تطبيقات عميلة.

| الحقل | الوصف | نوع البيانات | قواعد التحقق |
| :--- | :--- | :--- | :--- |
| `email` | البريد الإلكتروني الرسمي للدعم | String | صيغة بريد إلكتروني صالحة |
| `website` | رابط الموقع الإلكتروني الرسمي | String (URL) | يبدأ بـ `https://` |
| `github` | رابط الحساب أو المنظمة على GitHub | String (URL) | يبدأ بـ `https://` |
| `facebook` | رابط الصفحة الرسمية على فيسبوك | String (URL) | يبدأ بـ `https://` أو مصفوفة فارغة في حال عدم التوفر |
| `youtube` | رابط القناة الرسمية على يوتيوب | String (URL) | يبدأ بـ `https://` أو مصفوفة فارغة في حال عدم التوفر |
| `telegram` | رابط قناة أو بوت الدعم الفني على تيليجرام | String (URL) | يبدأ بـ `https://` أو مصفوفة فارغة في حال عدم التوفر |

## 9. `changelog.json`
* **وظيفة الملف:** الاحتفاظ بسجل التغييرات الكامل والتفصيلي والتاريخي للبرنامج، مصمم خصيصاً ليتم قراءته من قبل البرنامج نفسه لعرض شاشة "ما الجديد" (What's New) للمستخدمين، أو عرضه بتبويب مستقل بالموقع.
* **الجهة المستهلكة:** كلا الطرفين (البرنامج والموقع الرسمي).
* **البنية الأساسية:** كائن يحتوي على الحقول التعريفية وحقل `"data"` الذي يحوي مصفوفة التغييرات التاريخية.

### حقول عناصر مصفوفة `data`:
* `version` (String): رقم الإصدار المطروح (مثل: `2.0.2`).
* `buildNumber` (Integer): رقم البناء المقابل للإصدار لتسهيل المقارنة الرياضية.
* `releaseDate` (String - Date): تاريخ الإصدار بالصيغة القياسية `YYYY-MM-DD`.
* `summary` (String): ملخص إجمالي ومختصر جداً للتحديث الحالي.
* `changes` (Object): كائن يحتوي على تفاصيل التعديلات مقسمة إلى المصفوفات التالية (Array of Strings):
  * `additions` (مصفوفة نصوص): الميزات الجديدة التي تمت إضافتها.
  * `improvements` (مصفوفة نصوص): التحسينات البرمجية التي طرأت على الواجهات والأداء.
  * `fixes` (مصفوفة نصوص): المشاكل والأخطاء البرمجية التي تم إصلاحها.
  * `security` (مصفوفة نصوص): التحسينات الأمنية وحماية خصوصية وأكواد المستخدمين.
